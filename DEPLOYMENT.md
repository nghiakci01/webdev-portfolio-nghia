# 🚀 InfinityFree Deployment Guide

Follow these steps to deploy your portfolio to **InfinityFree**.

## Prerequisites
1.  An **InfinityFree** account.
2.  An FTP client like **FileZilla** OR use the **Online File Manager** in the hosting control panel.
3.  Your **EmailJS** credentials (optional but recommended for the contact form).

## Step 1: Configure Environment Variables
1.  Open [`.env.production`](file:///c:/laragon/www/webdev-portfolio-nghia/.env.production).
2.  Fill in your actual **EmailJS** keys.
3.  Set `VITE_SITE_URL` to your InfinityFree domain (e.g., `https://yourname.infinityfreeapp.com`).

## Step 2: Build the Project
Run this command in your local terminal:
```bash
npm run build
```
This will create a `dist/` folder containing all the optimized static files.

## Step 3: Upload to InfinityFree
1.  Connect to your InfinityFree server via FTP or use the Online File Manager.
2.  Navigate to the `htdocs/` folder (this is the root directory for your website).
3.  **Delete** the default `index2.html` file (if it exists) in `htdocs/`.
4.  Upload **all contents** from your local `dist/` folder into the `htdocs/` folder on the server.
    *   *Note: Do NOT upload the `dist` folder itself, only its contents (assets/, index.html, .htaccess, etc.)*

## Step 4: Verify Routing
The [`.htaccess`](file:///c:/laragon/www/webdev-portfolio-nghia/public/.htaccess) file is already included in the build. This ensures that React Router works correctly when you refersh the page on a sub-route.

## Troubleshooting
- **404 error?** Check if `.htaccess` was uploaded correctly to the root of `htdocs/`.
- **Contact form not working?** Ensure you filled in the keys in `.env.production` **BEFORE** running `npm run build`.
- **Images not showing?** Vite handles image paths automatically, but make sure the `assets/` folder is fully uploaded.
